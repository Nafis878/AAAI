# Building the paper

No LaTeX toolchain exists on the experiment machine, so the manuscript ships as
source. Two ways to build:

## 1. With the official AAAI-27 author kit (required for submission)
1. Download the kit from https://aaai.org/authorkit27/ (contains `aaai2027.sty`,
   `aaai2027.bst`, `ReproducibilityChecklist.tex`, sample sources).
2. Drop `aaai2027.sty` and `aaai2027.bst` next to `main.tex`.
3. Build: `pdflatex main && bibtex main && pdflatex main && pdflatex main`
   (or `latexmk -pdf main`).
4. Append the kit's reproducibility checklist after the references (it does not
   count against the 7-page main-content limit).
5. Submission variant: keep `\usepackage[submission]{aaai2027}` (anonymous);
   camera-ready uses the kit's `CameraReady` option instead.

## 2. Quick preview without the kit
Replace the `\usepackage[submission]{aaai2027}` line with
`\usepackage{times}` and build with any pdflatex; layout will not be
AAAI-conformant but content and figures render.

## Figures
All PDFs in `figures/` are generated by `python -m src.figures --all` from raw
run data in `experiments/` — regenerate before building if runs changed.

## Compliance
Validate the built PDF against the checklist in
`research/day1/aaai_requirements.md` §8 before submission (page limits,
anonymity, fonts, AI-use statement, reproducibility checklist).
